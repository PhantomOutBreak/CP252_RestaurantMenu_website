#**ที่มาและความสำคัญ**

  

หลายคนพบปัญหาที่จำเมนูอาหารที่เคยทำหรือเคยรับประทานไม่ได้ ทำให้เสียเวลาในการค้นหาข้อมูลใหม่หรือคิดเมนูซ้ำ ๆ

เว็บไซต์นี้มีจุดมุ่งหมายเพื่อช่วยให้ผู้ใช้งานสามารถดูและเรียกคืนข้อมูลเมนูอาหารที่เคยทำหรือสนใจได้อย่างสะดวกและรวดเร็ว ลดความยุ่งยากในการคิดเมนูใหม่

  

---

#**จุดประสงค์**

  

เว็บไซต์นี้ถูกสร้างขึ้นเพื่อช่วยให้ผู้ใช้งานสามารถดูเมนูอาหารที่อาจจะลืมหรือไม่ค่อยได้ใช้บริการบ่อย ๆ โดยเฉพาะอย่างยิ่งในกรณีที่ต้องการค้นหาสูตรอาหารหรือเมนูที่เคยทำหรือเคยสนใจไว้ในอดีต เว็บไซต์จะทำการจัดเก็บข้อมูลเมนูเหล่านั้นและแสดงผลให้ผู้ใช้งานสามารถเรียกดูได้อย่างสะดวกและรวดเร็วโดยไม่จำเป็นต้องมีการปรับแต่งหรือแก้ไขข้อมูลใด ๆ ทั้งสิ้น

  

---

#**ผลที่คาดว่าจะได้รับ**

การใช้งานเว็บไซต์นี้ไม่เพียงแค่ช่วยให้ผู้ใช้งานค้นหาเมนูที่เคยทำมาก่อน แต่ยังช่วยให้การวางแผนมื้ออาหารในอนาคตสะดวกขึ้น ด้วยการให้ข้อมูลเมนูที่หลากหลายและสามารถเข้าถึงได้ง่ายทุกเมื่อที่ต้องการ นอกจากนี้ เว็บไซต์ยังอาจมีฟีเจอร์ช่วยบันทึกเมนูโปรดหรือรายการที่ต้องการลองในอนาคต ทำให้ผู้ใช้งานไม่พลาดเมนูเด็ด ๆ ที่อยากลองทำในครั้งถัดไป

#**Functional และ Non-Functional**
  
- Functional

ผู้ใช้สามารถค้นหาเมนูอาหารได้

แสดงข้อมูลส่วนประกอบและวิธีการทำอาหาร

มีหมวดหมู่เมนู เช่น อาหารคาว อาหารหวาน หรืออาหารเพื่อสุขภาพ

ผู้ใช้สามารถบันทึกเมนูโปรดไว้ดูภายหลังได้

ระบบแสดงภาพตัวอย่างของเมนูอาหาร

- Non-Functional Requirements

เว็บไซต์ต้องโหลดข้อมูลภายในไม่เกิน 3 วินาที

รองรับการใช้งานบนอุปกรณ์มือถือและเดสก์ท็อป (Responsive Design)

ความปลอดภัยของข้อมูลผู้ใช้งาน

อินเทอร์เฟซใช้งานง่ายและไม่ซับซ้อน

#**DataStructure**
- จัดเก็บรายการเมนูอาหารทั้งหมด

**Array และ List**

เมนูอาหารเป็นรายการที่มีโครงสร้างคล้ายกัน (ชื่อ, รูปภาพ, วัตถุดิบ, วิธีทำ)

ใช้งานง่ายเมื่อดึงข้อมูลทั้งหมดเพื่อแสดงผล

ค้นหาด้วยชื่อเมนูผ่าน Search Bar

- จัดหาเมนู

**Hash Map และ Dictionary**

เวลาใช้ Search Bar สามารถค้นหาชื่อเมนูได้เร็วขึ้นถ้าใช้ Hash Map)

สามารถดึงข้อมูลของเมนูที่ผู้ใช้กด Favorites หรือ To-Do ได้ง่าย

-ระบบจัดเก็บเมนูที่ผู้ใช้กด Favorites, Done, To-Do-

Set ไม่มีข้อมูลซ้ำ ทำให้มั่นใจว่าเมนูเดียวกันจะไม่ถูกเพิ่มซ้ำ

การเพิ่ม ลบ หรือค้นหาข้อมูลใน Set

การใช้setเพื่อจัดรียงว่าเมนูไหนทำแล้วและเมนูไหนต้องทำ

- User Profile

**Object,Set,Hash Map**

Hash และ Objectจัดเก็บข้อมูลโปรไฟล์ของผู้ใช้แต่ละคน

setใช้รายการโปรด (favorites), ทำแล้ว (done), To-Do (toDo)
